= MailCatcher Helm Chart
Brian Matthews <brian.matthews@buralotech.com>
:icons: font
:chart-version: 0.2.0

This repository contains the artifacts required to build a Helm Chart for https://mailcatcher.me/[MailCatcher] allowing you to run a simple SMTP server for testing purposes in your development and testing environments.

== Prerequisites

* Kubernetes 1.23+

* Helm 3.8.0+

== Usage

You can consume this chart in any of the following ways:

* Install the chart with the release name `mc-release`:
+
[source,console]
----
$ helm install mc-release oci://registry-1.docker.io/bmatthews68/mailcatcher
----

* You can download the chart with to the current directory as `mailcatcher-{chart-version}.tgz`:
+
[source,console,subs="+attributes"]]
----
$ helm pull oci://registry-1.docker.io/bmatthews68/mailcatcher --version {chart-version}
----

* Add as a dependency to another chart:
+
[source,yaml,subs="+attributes"]
----
dependencies:
  - name: "mailcatcher"
    version: "{chart-version}"
    repository: "oci://registry-1.docker.io/bmatthews68"
----

== Ingress

Ingress via https://kubernetes.io/docs/concepts/services-networking/ingress-controllers/[Ingress Controllers] is supported since _v0.1.0_.
But it is only since _v0.2.0_ that it was possible to override the context path.

You can install the Helm chart with using a root context path:

[source,console]
----
helm install mail chart --set ingress.enabled=true,ingress.className=nginx
----

or, you can override the context path by setting the `pathPrefix`:

[source,console]
----
helm install mail chart --set ingress.enabled=true,ingress.className=nginx,pathPrefix=/inbox
----

== Kubernetes Gateway API

Ingress via https://kubernetes.io/docs/concepts/services-networking/gateway/[Kubernetes Gateway API] is supported since _v0.2.0_.

First you need to create a *Gateway* resource if the *Kubernetes Gateway API* implementation does not already provide one:

[source,console]
----
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: gateway # <1>
spec:
  gatewayClassName: istio # <2>
  listeners:
    - name: http
      hostname: "mail.local" # <3>
      protocol: HTTP
      port: 80
EOF
----
<1> This should match the value specified for `httpRoute.parentRefs[0].name`. The default is `gateway`.
<2> Depends on the installed Service Mesh or API Gateway e.g. `istio` for https://istio.io/[Istio].
<3> This should match the value specified for `httpRoute.hostnames[0]`. The default is `mail.local`.

You can install the Helm chart with using a root context path:

[source,console]
----
helm install mail chart --set httpRoute.enabled=true
----

or, you can override the context path by setting the `pathPrefix`:

[source,console]
----
helm install mail chart --set httpRoute.enabled=true,pathPrefix=/inbox
----
